GitHubってなに？

作ったものをしまって、なおして、そしてウェブで見せることもできる。
GitHubは、ものづくりのためのとても便利な場所です。

ひとことで言うと

GitHubは、自分が作ったファイルをしまっておける場所であり、 作ったものを少しずつよくしていく記録を残せる場所でもあります。

本来はプログラムを作る人たちがよく使う道具ですが、 下諏訪Labではもっとシンプルに、 「自分で作ったHTMLを、ウェブで見られるようにする場所」として使っています。

たとえば、こんな場所だと思えばOK

本だな

自分が作ったファイルを、なくさないように入れておける場所です。

作業ノート

どこを直したか、前はどうだったかを、あとから見返せます。

発表台

HTMLをアップすれば、ウェブページとして見せられることがあります。

GitとGitHubのちがい

名前が少し似ていますが、役わりは少しちがいます。

Git

ファイルの変更を記録していくためのしくみです。
「どこを直したか」を残していくイメージです。

GitHub

Gitで記録したものや、作ったファイルをネット上で管理する場所です。

下諏訪Labでの使い方

むずかしいこと全部よりまず、作ったHTMLを置いてウェブに出す使い方に集中します。

ここではこう考えれば十分

Gitは「直した記録を残すしくみ」、 GitHubは「そのファイルを置いておく場所・見せる場所」。 まずはこの理解で大丈夫です。

下諏訪Labでは、どう使っているの？

下諏訪Labでは、AIにお願いして作ったHTMLファイルを、 GitHubに入れてウェブで見られるようにしています。

流れのイメージ
  • やりたいことを言葉で考える
  • AIに「こういうページを作って」とお願いする
  • HTMLファイルができる
  • そのHTMLをGitHubに入れる
  • ウェブで開いて見られるようになる

ここが安心ポイント

特別なアプリを1から作らなくても、 HTMLファイル1つでも、ちゃんと自分の作品としてウェブに出せます。 「こんなの本当に出せるのかな」と思うものでも、 まずは小さく作って試せるのがいいところです。

ウェブに出すって、そんなに難しいの？

本格的な開発は難しいこともあります。けれど、 HTMLファイルを1つ作って見せるというところなら、 思っているよりずっと手が届きやすいです。

HTMLを作る ページの中身を作ります。文字、色、ボタン、画像などを入れられます。
GitHubにアップする 自分のリポジトリという場所に、そのHTMLファイルを入れます。
公開して見る 設定をすると、ブラウザで見られるページになります。

最初から全部わからなくて大丈夫

大事なのは、言葉で作りたいものを考えられることです。 むずかしい操作は、少しずつ覚えれば十分です。

GitHubのよいところ

すぐ直せる

文字を変えたり、見た目を直したりして、またすぐ反映できます。

前の状態も見やすい

前はどうだったかを見返しやすいので、こわがらずに直しやすいです。

人に見せやすい

「見てみて」と言える形で、自分の作品を外に出せます。

つまり、GitHubは何のために使うの？

下諏訪Labで大事にしているのは、 「考えたものを、ちゃんと形にして、見えるところまで持っていくこと」です。

GitHubは、そのためのとても便利な道具です。 プログラマーだけの特別な場所ではなく、 自分のアイデアを作品として出すための場所として使えます。

作れるだけじゃなく、見せられる

AIで作ることができる。
そしてGitHubを使えば、それをウェブで見せることもできる。

「自分の考えたものが、ほんとうに形になるんだ」 という感覚を持てることが、とても大きな一歩です。
